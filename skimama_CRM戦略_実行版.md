## 0. 要約（エグゼクティブサマリー）

- skimama卒業生の職種内訳（秘書50%／ライター23%／SNS運用17%／デザイン9%、CLAUDE.md参照）は、そのまま**リード段階での意欲・自己効力感のばらつき**を映していると仮定する。オンライン秘書志望が多数派＝「ビジネス」という言葉自体に距離がある未経験・主婦層がボリュームゾーンであり、CRM全体のデフォルト設計（トーン・語彙・不安の扱い方）はこの層に合わせる。
- 一方でInstagram運用代行・デザイナー志望層は、絶対数は少ないが**既に「専門スキルで稼ぐ」というビジネス的な発想に片足を入れている**、意思決定速度が速い高確度層。同じトーン・同じ配信速度で扱うと、ボリューム層には情報過多・専門層には物足りなさという両方の機会損失が起きる。
- したがって本設計の核は、**登録直後の1問診断でリードを2〜3トラックに分岐させ、以降のステップ配信・訴求・接触速度をトラックごとに変える**こと。トラックをまたぐ再判定（行動ログによる乗り換え）も許容する可変設計にする。
- L-stepの機能（回答フォーム／タグ／シナリオ分岐／スコアリング／リッチメニュー出し分け）はこの分岐設計を実装するための手段として位置づけ、§2で機能とskimamaの使い道を対応させる。

## 1. CRM設計の全体方針

### 1-1 なぜ「定性→データ」の順で進めるか

- L-stepは配信後に開封率・タップ率・回答率が全てログとして残るツール。**先に分岐ロジックとメッセージの型を決めておかないと、データが溜まっても「何が原因で数字が動いたか」を切り分けられない**（トーンが悪いのか、分岐が粗いのか、配信間隔が悪いのかが混ざる）。
- 本書は「初期仮説シナリオ」を確定させることが目的。運用開始後は§12のPDCA設計に沿って、トラック別の開封率・タップ率・予約率を比較し、仮説を棄却・更新していく。

### 1-2 CRMが解くべき問題（CLAUDE.md行動コスト3層モデルの接続）

CLAUDE.mdの整理どおり、LINE登録した時点の相手は「行動コスト層2→層3」の境界にいる（すでに何か行動を起こした人）。CRMの仕事は比較検討・値踏みだけではなく、**層3（新しい行動＝入会）に踏み切れない残りのハードルを、トラックに応じて的確に取り除くこと**。

| トラック | 層3に踏み切れない主な理由（仮説） | CRMが埋めるべきギャップ |
|---|---|---|
| オンライン秘書・未経験ボリューム層 | 「自分にPCスキルなんてない」「ビジネス用語が分からない時点で向いていない気がする」という自己効力感の低さ | 専門用語を使わず、"あなたと同じ人ができた"実例を反復し、心理的ハードルを下げる |
| IG運用代行・デザイナー志望層 | 「本当にこのスクールで案件が取れる実力がつくのか」「独学や他スクールとの比較で分が悪くないか」という実力・費用対効果の見極め | スキルの中身・案件相場・ポートフォリオ・卒業生の専門性の高さを具体的に見せ、比較検討に正面から答える |

## 2. L-stepの機能棚卸しとskimamaでの使い道

L-stepの主要機能を、skimama CRMのどの役割に当てるかを先に定義する（実装担当者がそのままタスクリストに変換できる形）。

| L-step機能 | skimamaでの使い道 |
|---|---|
| **回答フォーム（アンケート）** | 登録直後の「職種トラック診断」。回答内容をそのままタグ付けトリガーにする（§4） |
| **タグ管理** | ①職種トラック（秘書／SNS運用／デザイン／ライティング／未定）②現在の就業状態（専業・パート・育休中・その他）③関心の強さ（診断回答＋クリック行動の累積）④ファネル進捗（登録済／勉強会予約済／参加済／未参加／入会検討中）を掛け合わせて管理 |
| **シナリオ配信（ステップ配信）** | トラック×進捗タグの組み合わせで配信内容を分岐（§8） |
| **流入経路別配信・URL計測** | Organicのどの投稿・どのプラットフォーム経由の登録かをタグ化し、初期メッセージの温度感を微調整（例：デザイン系リール経由の登録者はデザイントラックへの誘導を診断より前に軽く提示） |
| **クリック計測によるタグ追加** | 診断で「まだわからない」と回答した層を、その後どのリンク（秘書の実例／デザインの実例等）を繰り返しクリックしたかで後追いトラック判定する（§4-4） |
| **スコアリング** | 開封・タップ・回答・動画視聴等の行動に点数を設定し、一定点数超で有人フォロー対象として抽出（§11） |
| **リッチメニュー出し分け** | トラック別に文言・導線を出し分け（§10） |
| **配信予約・時間帯最適化** | 主婦層は日中〜夕方の隙間時間、専門職層は夜間の集中時間帯など、トラック別に配信時間帯を仮説設定し検証（§12で検証） |

## 3. トラック分岐の全体設計

### 3-1 分岐点はどこに置くか

分岐は「登録直後の自己申告」だけに頼らない。**申告と行動ログの2段構えで判定精度を上げる**（自己申告だけだと「まだわからない」「秘書と答えたが実はデザインに興味がある」という取りこぼしが必ず出るため）。

```
① 登録直後（Day 0）：1問診断フォームで一次タグ付け
   └─ 「今、一番気になる働き方はどれですか？」
      選択肢：①オンライン秘書・事務系　②Webライティング
             ③デザイン　④Instagram運用代行　⑤まだ決めていない・全部気になる

② Day 0〜7：行動ログによる二次判定（補正）
   └─ どの職種の卒業生実例リンクを開いたか／どの配信の滞在が長いかでタグを補正
      「まだ決めていない」層は原則トラックA（ボリューム層仕様）をデフォルト適用しつつ、
      クリック傾向が専門職に偏れば途中でトラックBへ乗り換え

③ 勉強会予約・参加後：本人の発言・アンケート回答で最終確定
   └─ 勉強会後アンケートで職種希望を再確認し、クロージング期の訴求を確定させる
```

### 3-2 3トラック構成（デフォルト＝トラックA）

| トラック | 対象 | 想定ボリューム（卒業生職種内訳からの仮説） | 位置づけ |
|---|---|---|---|
| **トラックA：ビギナー・生活密着型** | オンライン秘書志望が中心。未経験・専業主婦／パート層、「ビジネス」という言葉に距離がある層 | 最大ボリューム（卒業生内訳の秘書50%が示唆） | **CRM全体のデフォルト仕様**。診断未回答・「まだ決めていない」もここに初期配属 |
| **トラックB：専門職・高確度型** | Instagram運用代行・デザイナー志望。副業／フリーランス的発想に片足を入れている層 | 少数だが意思決定速度が速い（内訳17%＋9%） | 訴求の専門性・比較検討への回答を厚くする |
| **トラックC：ハイブリッド（ライティング）** | Webライティング志望。「未経験だが言葉には自信がある」層で、A・Bの中間的な心理状態 | 内訳23%と無視できない規模 | 基本はトラックAのトーンをベースに、実務的な訴求（文字単価・案件の具体性）をBから借用するハイブリッド設計（詳細は§7） |

> **要検証**：この3トラックの人数配分・トーン設計は「卒業生の最終職種内訳」からの逆算仮説にすぎず、**入会前のリード段階での初期関心と一致するとは限らない**（例：秘書に配属された卒業生の一部は、入会前はデザインに興味があったが選考・適性で秘書になった可能性がある）。運用開始後、診断回答の分布と最終職種の突合データが取れ次第、トラック定義そのものを見直すこと（§13）。

### 3-3 なぜオンライン秘書層をデフォルトにするか

- 母数が最大であるため、**設計ミスの影響範囲が最も大きい**。デフォルト＝最も丁寧に作り込むべき対象という意味で置く。
- 「まだ決めていない」「複数選んだ」「診断に回答しなかった」離脱者・保留者を、専門用語の多いトラックBのトーンに誤って乗せると離脱リスクが上がる。**判断に迷う場合は易しい方に倒す**のが安全側の設計。

## 4. トラックA：ビギナー・生活密着型の設計

### 4-1 ペルソナ像（CLAUDE.md Tier1/Tier2の翻訳）

「PCは苦手ではないがWordとメールくらい」「"オンライン秘書"という言葉も実は勉強会で初めて知った」「"Webライター""SNS運用代行"と言われてもピンとこない」「自分が"働く"というより"誰かの役に立つことを手伝う"イメージなら想像できる」というレベル感。ビジネス誌的な語彙（KPI、ポートフォリオ、単価交渉、案件受注、etc.）は原則使わない。

### 4-2 トーン原則

| 原則 | 具体例 |
|---|---|
| 専門用語を使わない／使う時は必ず言い換える | ×「まずはポートフォリオを作成」→○「まずは"できます"を証明する実例を1つ作る」 |
| 「あなたにもできる」を否定形で裏付ける | 「特別なスキルは要りません」「PCが得意でなくても大丈夫です」を具体的な卒業生像とセットで出す（断定表現は§13コンプライアンス参照） |
| 小さな一歩を毎回具体的に示す | CTAは常に「動画を1本見る」「LINEで1つ質問に答える」など、5分未満で終わる行動に分解する |
| 失敗・不安への先回り共感 | 「"今さら"と思っていませんか」「"続けられるか不安"なのは皆同じです」と本人の想定質問を配信側から先に言語化する |

### 4-3 主な不安と配信での回答

| 不安（層2〜3ペイン） | 配信での回答 |
|---|---|
| 「子供がいて決まった時間に学習できない」 | 週4動画＋2日課題＋1日ワークショップの柔軟性、隙間時間学習の卒業生の声 |
| 「そもそも"オンライン秘書"が何をする仕事か分からない」 | 業務内容を家事・育児の延長でイメージできる言葉に翻訳（スケジュール調整＝家族の予定管理の延長、等） |
| 「稼げる自信がない」 | 断定は避け、獲得率93%の根拠と算出条件を明示した上で「同じような状態だった人の実例」を反復提示 |
| 「講座についていけるか不安」 | メンター伴走・少人数制・満足度99.3%を「一人にしない」という情緒面から説明 |

### 4-4 「まだ決めていない」層の扱い（行動ログ判定の起点）

初期タグは全員トラックAのトーンで配信しつつ、以下の行動が一定回数観測されたらトラックB候補として補正タグを付与し、次のステップからトーンを段階的に切り替える。

- デザイン・SNS運用系の卒業生実例リンクを複数回クリック
- 「案件相場」「ポートフォリオ」等、専門職向けコンテンツへの反応（開封・タップ）が秘書系コンテンツより明確に高い

## 5. トラックB：専門職・高確度型の設計

### 5-1 ペルソナ像

「フリーランス」「副業」「案件」という言葉に既に馴染みがある。Instagramを個人でも触ったことがある、デザインツール（Canva等）を趣味で触ったことがある、といった**準備状態が一段進んだ層**。意思決定において「本当にこのスクールで実力がつくか」「他スクール・独学との比較でどうか」を早い段階で気にする。

### 5-2 トーン原則

| 原則 | 具体例 |
|---|---|
| 専門性を隠さない | 「デザインの基本原則」「アカウント設計の考え方」など、トラックAより一段専門的な語彙をあえて使い、"本気で学べる場"であることを示す |
| 比較検討に正面から答える | 独学との違い、他スクール（Fammのデザイン特化・SHElikesの学び放題）との違いを早い配信段階で提示（01_戦略/01_競合リサーチ.md参照） |
| 実例は「実力」の証拠として見せる | 卒業生の成果物（デザイン作品・運用実績のビフォーアフター数値等）をトラックAより早いタイミングで、具体的に見せる |
| 意思決定速度に合わせて接触を早める | トラックAより配信間隔を詰める、勉強会案内を早期に出す（§8参照） |

### 5-3 主な検討ポイントと配信での回答

| 検討ポイント | 配信での回答 |
|---|---|
| 「独学やYouTubeでも学べるのでは」 | 独学の挫折率・迷子リスクを卒業生の実体験で言語化し、伴走の価値を提示 |
| 「Fammの方がデザイン特化で強いのでは」 | 単一スキル一点賭けのリスクと、skimamaが複数職種から選べる柔軟性・仕事獲得までの伴走の違いを提示（01_戦略/01_競合リサーチ.md §2） |
| 「本当に案件が取れるのか」 | 獲得率93%の根拠、フジ子さん等提携先の存在、卒業生の初案件までの実期間 |

## 6. （補助）トラックC：ハイブリッド・ライティング型

- ベースはトラックAのトーン（未経験前提・専門用語を避ける）を維持しつつ、「文字単価」「Webライティングの案件例」など**実務寄りの具体性はトラックBから部分的に借用**する。
- 診断で「Webライティング」を選んだ場合、Day 0〜7はトラックA相当の安心材料を提示し、Day 7以降（比較検討フェーズ）からトラックB相当の実務情報（案件例・単価感）を混ぜる、という**時間差のハイブリッド**が運用上シンプル。
- 独立トラックとして作り込みすぎるとL-step運用の複雑度が上がるため、初期構築では「トラックAシナリオ＋ライティング専用の差し込みブロック2〜3通」という最小構成から始める。

## 7. 共通シナリオ骨子とトラック別の出し分けポイント

brain/09_実務戦略パターン集.md §8の4ステップ型を土台に、トラック分岐点を明記して詳細化。

```
Day 0（登録直後）
  共通：ウェルカムメッセージ＋在宅ワークママBOOKの受け渡し
  共通：職種トラック診断（1問フォーム）→ タグ付け・トラック仮確定
  分岐：トラックA/C＝「特別なスキルは不要です」の安心メッセージを追加
        トラックB＝「あなたの興味に近い卒業生の実例」を1件先出し

Day 1〜3
  分岐：トラックA/C＝日常に寄せたペイン共感コンテンツ（パートの時給換算・
        送迎の合間時間の使い方等）
        トラックB＝スキル訴求コンテンツ（案件相場・ポートフォリオの考え方）

Day 3〜5（勉強会案内）
  分岐：トラックA/C＝「子供がいても大丈夫」等、参加ハードルを下げるFAQを厚めに
        トラックB＝日程・当日の学べる内容の専門性を先に明示し、早めにCTAを出す
        （意思決定速度が速いため、Aより早いタイミングで予約導線を強める）

Day 7前後（勉強会参加者へ）
  共通：獲得率93%等の根拠提示
  分岐：トラックA/C＝「あなたと同じ状態だった卒業生」の情緒的ストーリー中心
        トラックB＝専門性・案件獲得の具体プロセス中心

Day 10〜14（比較検討・クロージング期）
  分岐：トラックA/C＝不安払拭FAQ、分割払い（月12,000円〜）の提示、個別相談導線
        トラックB＝競合（Famm・SHElikes・独学）との比較コンテンツ、実力担保の証拠

Day 14〜20（未申込者フォロー）
  共通：未参加・未申込者への再アプローチ
  分岐：トラックA/C＝「まだ迷っていて大丈夫」という背中押し＋個別相談の軽い提案
        トラックB＝期間限定の個別相談・質問受付など、意思決定を後押しする具体アクション

Day 20〜25（離脱防止・最終フォロー）
  共通：未行動者への最終アプローチ（限定特典 or 個別相談）
  分岐：この段階で反応がない場合はトラックを問わず「無関心期に戻す」タグへ移行し、
        頻度を落とした低温メンテ配信（月1回程度の有益コンテンツ）に切り替える
```

> 配信間隔・日数は仮説。トラックBは意思決定が速い可能性が高いという仮定のもと接触を早めているが、実データで「むしろ検討期間が長い」等の逆の傾向が出れば§12で修正する。

## 8. タグ設計表（L-step実装用）

| タグ分類 | タグ例 | 付与トリガー |
|---|---|---|
| 職種トラック | `trk_A_秘書` / `trk_B_SNS運用` / `trk_B_デザイン` / `trk_C_ライティング` / `trk_未定` | 診断フォーム回答（初期）／行動ログ補正（§4-4） |

## 12. 表現コンプライアンス（景表法・特商法）

CRMはLPや広告以上に**1対1の会話に近い体裁**になるため、断定表現が個人向けの「保証」と受け取られやすいリスクがある。特にトラックBでは「案件が取れる」「稼げる」に踏み込みたくなる場面が増えるため注意。

| NG表現 | OK言い換え |
|---|---|
| 「必ず案件が取れます」「誰でも稼げます」 | 「在宅ワーク獲得率93%（算出条件：〇〇）という実績があります」＋個別の実例提示 |
| 「〇〇円稼げます」という断定的な収入提示 | 「卒業生の中には月〇万円の案件を受注した方もいます（個人の実績であり保証するものではありません）」 |
| 診断結果を「あなたは絶対デザイナー向き」等、断定的に適性診断化する | 「〇〇に関心が高い傾向が見られます」という提示にとどめ、最終選択は本人に委ねる文言にする |
