## 0. 前提：この企画書が土台にしている結論

この企画書はゼロから発想したものではなく、以下の既存結論の**実行翻訳**である。個別企画を追加・修正する際も必ずこの前提に立ち返る。

| 土台 | 参照元 | この企画書での使い方 |
|---|---|---|
| **JTBD**：機能（在宅で収入源を作る）・情緒（"何かができる自分"を取り戻す）・社会（"ちゃんと前に進んでいる母親"を示す）の3つの仕事 | 社内の顧客理解の核／戦略書 §3-1 | 全企画に「どのJTBDを動かすか」をタグ付けする。機能的仕事だけの企画はスクール比較の土俵に乗るため、情緒・社会を必ず混ぜる |
| **ペイン4層**：表層（在宅で稼ぎたい）→顕在（パート代が保育料で消える）→潜在（社会から切り離された感覚・扶養への引け目）→インサイト（「自分の人生をまだ諦めていないと証明したい」） | 社内資料／戦略書 §3-2 | フックは層2で「これ自分だ」を作り、本文で層3を言語化して「分かってくれてる」を作る。層4は直接言わず、視聴者に言わせる |
| **行動コスト3層**：最大の競合は他スクールではなく「何もしない（層1）」「独学・ママ友副業（層2）」 | 社内資料／戦略書 §3-3 | Organicコンテンツの大半は層1・層2向け（市場を耕す）。競合スクール比較（層3向け）は少数精鋭にする |
| **行動変容ステージ×ライフイベント**：無関心期〜維持期。入園・育休復帰・小1の壁が検討の山を作る（季節性は仮説） | 戦略書 §5 | プラットフォーム役割分担と月次カレンダーの背骨 |
| **チャネル役割分担**：Organic＝需要創出・信頼の土台、広告＝刈り取り、LINE＝継続 | 戦略書 §6 | Organicに刈り取りKPIを直接負わせない。ただしLINE登録への導線は常設する |
| **ポジショニング一文**：「意識が高い系」でも「怪しい副業」でもない、同じ"普通のママ"が伴走してくれる在宅ワークの実家 | 戦略書 §4 | 全アカウント・全投稿のトーンの判定基準。「キラキラ」「煽り」に寄ったら差し戻す |

> **この企画書のゴール**：フォロワーを増やすことではなく、**無関心期〜準備期のママを一段ずつ動かし、LINE登録・無料勉強会予約という「ゼロリスクの一歩」まで運ぶ持続的な導線**を作ること。

## 1. KPIツリー（最終KPIからの逆算）

> **原則：フォロワー数を目的化しない**（フォロワーが伸びるかは9割投稿の質で決まる。フォロワーは結果指標）。操作できるのは投稿の質＝滞在時間・保存・コメントであり、事業に効くのはLINE登録・勉強会予約である。

```
最終KPI：LINE登録数・無料勉強会予約数（Organic経由）
│
├─ 中間KPI（チャネル共通）
│   ├─ プロフィールアクセス数 × プロフィール→LINEリンクタップ率
│   ├─ ストーリーズ/固定投稿からのLINE遷移数
│   └─ （YouTube）概要欄・コメント固定からのLINE遷移数
│
├─ 先行KPI（アルゴリズム上の操作変数）
│   ├─ 平均視聴維持率・滞在時間（最重要）
│   ├─ 保存率（「あとで見返したい」＝教育コンテンツの質）
│   ├─ コメント数（誘発設計の質）
│   └─ リーチ→プロフィールアクセス率（フックとプロフィール導線の質）
│
└─ 資産KPI（長期）
    ├─ フォロワー数・フォロワーのターゲット純度（結果指標として観測のみ）
    ├─ UGC数（卒業生発信・メンション数）
    └─ 指名検索数・「skimama」タグ付き投稿数
```

**運用ルール**：
- 週次で見るのは先行KPI（維持率・保存・コメント）。月次で中間KPI（LINE遷移）。四半期で最終KPI・資産KPI
- 「伸びない」の原因診断はアルゴリズム要因→企画品質の順で行う（初動10〜20本はジャンル認識前で伸びないのが正常。それ以降はクオリティ問題として企画・フック・テンポを疑う）
- **現状のベースライン（各アカウントのフォロワー・維持率・LINE登録数）が未取得**。参画後最初のタスクとして取得し、本ツリーに現状値→3ヶ月目標→12ヶ月目標を書き込む（現状値未取得ならそれ自体を最初のタスクにする）

## 2. プラットフォーム別の役割分担

> **原則：同一コンテンツの全プラットフォーム一括横展開はしない。** 素材（卒業生ストーリー・企画の核）は共通化して制作コストを下げつつ、**プラットフォームごとに「狙うステージ×JTBD×フォーマット」を変えて出し分ける**。

| PF | 機能的役割 | 主な狙いステージ | 主なJTBD | 理由（アルゴリズム・ユーザー層特性） |
|---|---|---|---|---|
| **TikTok** | **新規リーチの最大化装置**。フォロワー外配信が前提のため「まだskimamaを知らないママ」への初回接触に最適 | 無関心期→関心期 | 情緒（共感・あるある） | レコメンド型でフォロワーゼロでも拡散する。ママ層の利用も拡大中（仮説・要確認）。ここで深い教育はせず「これ自分だ」の3秒を量産する |
| **Instagram（リール＋フィード＋ストーリーズ）** | **総合ハブ＋LINE導線の本丸**。リールで新規、フィード（カルーセル）で保存型教育、ストーリーズで日常・双方向、プロフィールがLINE登録への関所 | 関心期→準備期（全ステージの結節点） | 機能＋情緒＋社会すべて | 滞在時間・保存・コメントの複合評価。プロフィール→ハイライト→LINEという導線設計ができる唯一のPF。ママ層の情報収集の主戦場（仮説） |
| **Threads** | **テキストでの共感・双方向性・「中の人」の体温**。リールでは載せきれない本音・葛藤・失敗談をテキストで置き、コメント欄で対話する | 無関心期〜関心期（＋行動期の不安払拭） | 情緒・社会 | テキスト主体で「盛らない本音」が伸びやすい。IGアカウントと連携しプロフィール回遊を生む。広告色が最も嫌われる場所なので売り込みは原則しない |
| **YouTube（長尺＋Shorts）** | **検討層への深い解説と検索受け皿**。「在宅ワーク 未経験 ママ」等の検索意図に答える10〜20分動画、勉強会の疑似体験、卒業生ロングインタビュー | 準備期→行動期 | 機能（判断材料）＋信頼 | 検索起点＋長尺で信頼構築に最適。259,800円の意思決定には「顔が見える長い説明」が必要（無形商材＝経験財の試食）。Shortsはリール素材の転用で認知も補完 |

**プラットフォーム間の導線設計**：

```
TikTok / YT Shorts（初回接触）→ Instagramプロフィール（関所）→ ハイライト・固定投稿（教育）
→ LINE登録（特典：在宅ワークママBOOK）→ 無料勉強会
Threads（共感・対話）⇄ Instagram（相互回遊）
YouTube長尺（検討の最終確認）→ 概要欄LINEリンク → 無料勉強会
```

**投資順序の仮説**（「受け皿を整えてから拡張する」原則準拠）：
1. まずInstagram（プロフィール・ハイライト・LINE導線という「受け皿」）を整える
2. 次にリール×TikTok同時運用で新規リーチを拡大
3. YouTubeは制作コストが高いため、卒業生インタビュー素材が一定数たまってから本格化（Shorts転用は初期から）
4. Threadsは低コストなので初期から並走（1日1〜3ポスト、リール企画の副産物として運用）

## 3. アカウントコンセプトの方向性

### 3-1 ジャンル選定：レッドオーシャン×掛け合わせ

- 「ママ×在宅ワーク×副業」はレッドオーシャン＝需要の証明（ブルーオーシャンは初心者NG、競合が多い市場こそチャンス）
- **HARMの法則での掛け合わせ**：A（キャリア）×M（お金）を土台に、**H（メンタル・自己肯定感）を差別化の掛け算**として乗せる。「稼ぎ方」アカウントは無数にあるが、「稼げるようになる過程で自分を取り戻す」を正面に置くアカウントは少ない（仮説・競合30〜50アカウント分析で要検証）
- R（家族・夫婦）は企画単位で使う（「夫に反対されたとき」「家族の反応が変わった」等はコメントが伸びやすいテーマと推定）

### 3-2 コンセプト公式：ジャンル×ストーリーテリング（ビフォー→アフター）

**バイアス状態と理想状態のギャップが大きいほど強い**。

| 型 | バイアス（ビフォー） | 理想（アフター） |
|---|---|---|
| 卒業生ストーリー型 | 「PCはメール程度・事務パート10年・自分に何もないと思っていた」 | 「在宅のオンライン秘書として自分の名前で仕事をしている」 |
| 運営・中の人型 | 「skimamaの中の人も元・普通のママ」（該当者がいれば） | 「ママの働き方を変える側になった」 |
| 密着企画型 | 「未経験ママの在宅ワークデビューまで、リアル密着」 | 学習開始〜初案件獲得までの過程を隠さず見せる |

> **注意**：アフターの表現は「月◯万円稼げるようになった」と断定的に一般化しない。**個人の実例として、時期・職種・稼働時間の文脈つきで語る**（→ §10 表現ルール）。

### 3-3 アカウント設計の4問への回答案

1. **誰に**：パート・専業・育休中で「このままでいいのかな」と思っている、意識高い系が苦手な普通のママ（Tier1中心）
2. **どんな発信**：在宅ワークのリアル（きれいごと抜き）＋同じ境遇のママの一歩の記録
3. **どんな問題を解決**：「何から始めればいいかわからない」「私にできる気がしない」
4. **どんな未来**：家にいながら自分の名前で稼ぎ、「ちゃんとやっている自分」を取り戻した状態

**プロフィール設計**：1行目に発信テーマ（例：「未経験ママの在宅ワークデビューを応援」）、箇条書きで実績（在宅ワーク獲得率93%※・卒業生◯名・ベビーテックアワード2024優秀賞）、最終行にLINE誘導（「▼無料の在宅ワークママBOOKはこちら」）。※実績数値の算出定義確認後に掲出（→§11）。

## 4. コンテンツの3訴求軸と企画バリエーション

戦略書 §7-2の3軸を実企画に展開する。**配分ルールの初期仮説：認知・共感45%／信頼・証拠35%／行動・転換10%／人間味（Threads的な日常）10%**（実績を見て四半期ごとに見直す）。

### 軸A：認知・共感（無関心期〜関心期／層1・層2の耕し）

| # | 企画名 | フック案（冒頭1〜3秒） | 構成の要点 |
|---|---|---|---|
| A-1 | パート時給の現実算 | 「パート代、時給に直したら◯円だった話」 | 保育料・ガソリン代・急な欠勤を引いた"実質時給"を試算して見せる。層2ペインの可視化。数字はテロップで大きく。※特定の職業を貶めるトーンにしない |
| A-2 | 子どもの発熱、また私が休む | 「『また休むの?』って言われた日のこと」 | ストーリー型。層2（欠勤の気まずさ）→層3（社会から取り残される感覚）へ降りる。解決は出さず共感で終える。コメント誘発：「同じ経験ある方いますか」 |
| A-3 | 「ママは毎日何してるの?」に固まった話 | 「子どもに聞かれて、答えられなかった」 | 層3（何者でもなくなった感覚）の言語化。社会的JTBDに直撃。売り込みゼロで保存・シェアを狙う |
| A-4 | 現状維持の10年後シミュレーション | 「今のまま10年経ったら、を計算してみた」 | 行動コスト3層の層1向け：放置コストの可視化。収入天井・子の成長・自分の年齢を淡々と並べる。煽らず事実の提示で終える |
| A-5 | 在宅ワークあるある（挫折編） | 「在宅ワーク、正直これがしんどい」 | あえてデメリットから入る誠実路線。「怪しい副業」との差別化＝信頼の種まき。ポジショニング一文（実家感）に忠実に |
| A-6 | 夜、子どもを寝かしつけた後の30分 | 「みんな、寝かしつけの後って何してる?」 | TPO変更：「勉強は寝かしつけ後の30分でいい」の前段。Threads向きの問いかけ型 |

### 軸B：信頼・証拠（準備期〜行動期／層3比較検討層の不安払拭）

| # | 企画名 | フック案 | 構成の要点 |
|---|---|---|---|
| B-1 | 卒業生ビフォーアフター図鑑 | 「事務パート歴10年だった彼女の、今の働き方」 | §8の型に沿った卒業生ストーリーの連載化。職種・稼働時間・期間を必ず添え、個人の実例として語る。skimama最大の武器 |

## 5. 9セルマトリクス：ターゲットTier × ファネル段階

行＝ターゲットTier、列＝ファネル（行動変容ステージを潜在/準顕在/顕在に圧縮）。**各セルに「目的＋主フォーマット＋代表企画（§4の番号）」**を割り当てる。

| | **潜在層（無関心期〜関心期入口）**<br>目的：認知・共感、商品を出さない | **準顕在層（関心期〜準備期）**<br>目的：教育・比較材料 | **顕在層（準備期〜行動期）**<br>目的：不安払拭・後押し |
|---|---|---|---|
| **Tier1**<br>育休中〜復職直前／パート収入天井層（最優先） | 復職・パートあるある共感リール（A-2, A-4）<br>主PF：TikTok・IGリール<br>型：ストーリー型 | 職種図鑑・独学との比較（B-2, B-5）<br>主PF：IGカルーセル・YouTube<br>型：啓蒙・比較検証型 | 勉強会の中身公開・卒業生の声（B-6, B-1, C-3）<br>主PF：IG・YouTube長尺<br>型：第三者の声型 |
| **Tier2**<br>専業主婦「そろそろ何か」層 | 自己肯定・「私は何者?」系共感（A-3, A-6）<br>主PF：Threads・IGリール<br>型：ストーリー型・問いかけ型 | 「在宅ワークって実際どう?」の実像解説（A-5, B-2）<br>主PF：IG・YouTube<br>型：啓蒙型 | ゼロリスクの一歩の提示（C-1, C-2）<br>主PF：IGストーリーズ・LINE<br>型：サービス紹介型（控えめ） |
| **Tier3**<br>将来層（妊娠前・独身）＋季節性の強い層（入園・小1の壁） | 「ママになっても働き方は選べる」将来設計系（A-4の将来版）<br>主PF：TikTok・Threads<br>型：啓蒙型 | ライフイベント逆算コンテンツ（C-4の教育版）<br>主PF：IG・YouTube検索<br>型：季節訴求型 | イベント直前の行動喚起（C-4）<br>主PF：IG・LINE・広告連携<br>型：季節訴求型 |

**運用ルール**：
- 週次の企画会議でこの表に投稿予定を置き、**セルの偏り（潜在層ばかり／顕在層ばかり）を可視化**する
- Tier3の顕在セルは広告（刈り取り）との連携枠。Organic単独で無理に刈り取らない
- 各セルの反応データ（維持率・保存・LINE遷移）をたまり次第セルごとに評価し、配分を調整する

## 6. 月次投稿カレンダーの型（季節性反映）

> **前提**：検討の季節性（入園・育休復帰・小1の壁）は**仮説**であり、月次申込データでの検証が未了。カレンダーは仮説ベースで組み、実データで補正する。

### 年間の山（仮説）

| 時期 | ライフイベント | 狙うべき動き |
|---|---|---|
| 9〜12月 | 4月入園の保活・入園準備期 | 「入園後の働き方」検討が始まる。Tier1/3向け教育を厚く。C-4系の逆算コンテンツ投入 |
| 1〜3月 | 育休復帰期限・年度末・入園直前 | 準備期の山（仮説）。信頼・証拠軸を増量、勉強会導線を強化。「4月から変わる」訴求 |
| 4〜5月 | 入園・入学直後、小1の壁の顕在化 | 生活が変わり時間が生まれる/問題が噴出する時期。共感軸で受け止め→関心期へ |
| 6〜8月 | 夏休み（可処分時間が減る） | 学習訴求は弱まると推定。共感・人間味・保存型（あとで読む）コンテンツ中心に切り替え |

### 月次の型（通常月・週次リズム）

| 曜日感覚 | 内容 | 軸配分の目安 |
|---|---|---|
| リール/TikTok：週3〜4本 | A軸2本＋B軸1〜2本（新規リーチの主力） | 通常月：A45%／B35%／C10%／人間味10% |
| フィード（カルーセル）：週1〜2本 | B軸の保存型教育（職種図鑑・比較・FAQ） | 季節の山ではC（行動喚起）を最大25%まで引き上げ、A軸を圧縮 |
| ストーリーズ：毎日1〜3枚 | 日常・質問箱・投稿の裏側・LINE導線の常設 | |
| Threads：毎日1〜3ポスト | リール企画の本音版・問いかけ・コメント対話 | |
| YouTube：月2〜4本（長尺）＋Shorts転用 | 卒業生インタビュー・職種解説・勉強会疑似体験 | |

- 月初に9セルマトリクス上へ当月の投稿を配置→月末にセル別実績レビュー、が月次PDCAの単位
- 初動期（Phase 1）は**投稿練習10本（非公開）→初期50〜100本の企画分析**を先に行う

## 7. 台本設計の運用ルール（8ブロック当てはめ）

DtoC物販前提の台本テンプレートを翻訳する際は、以下の読み替えを**標準ルール**とする：④商品コンセプト→「なぜskimama/この職種があるのか」、⑤USP・機能→「伴走・職種選択・獲得実績」、⑥使用シーン→「ママの1日の中の学習シーン」。ペイン層との対応：**②現実の言語化＝層2、③原因・教育＝層3、⑦ビフォーアフター＝情緒変化（自己効力感）**。

### 台本例1：A-1「パート時給の現実算」（認知・共感軸／Tier1潜在層）

| ブロック | 内容（セリフ・テロップ案） |
|---|---|
| ①フック | 「私のパート代、"実質時給"にしたら◯◯円でした」（テロップ大） |
| ②現実の言語化（層2） | 「時給1,050円。でも保育料引いて、ガソリン代引いて、子どもの発熱で月2回休んで…」 |
| ③原因・教育（層3） | 「実は問題は時給じゃなくて、**時間を切り売りする働き方だと、何年働いても積み上がらない**こと」 |
| ④コンセプト | 「だから"時間を売る"から"スキルで働く"に変えたママたちがいます」 |
| ⑤USP | 「未経験から2ヶ月、秘書・ライター・SNS運用・デザインから自分に合う職種を選べて、仕事獲得まで伴走」 |
| ⑥シーン | 「学習は寝かしつけ後の30分から。動画学習だから途中で止められる」 |
| ⑦ビフォーアフター | 卒業生実例：「パートを辞めたことより、『私にもできた』って思えたことが大きかった」（実例・本人の言葉として） |
| ⑧クロージング | 「まず知るだけなら無料。プロフィールのLINEから在宅ワークママBOOKを受け取ってください」 |

### 台本例2：B-5「独学2年 vs 伴走2ヶ月」（信頼・証拠軸／Tier1準顕在層）

| ブロック | 内容 |
|---|---|
| ①フック | 「独学2年、収入ゼロだった私の敗因を話します」（卒業生本人出演が理想） |
| ②現実の言語化 | 「YouTubeで勉強→教材を買う→また別の教材→気づいたら2年。何者にもなれてない」 |
| ③教育 | 「独学の敗因はサボりじゃなくて、**"何をやらないか"を決めてくれる人がいない**こと」 |
| ④コンセプト | 「skimamaは職種を先に決めて、そこに要るものだけを2ヶ月で」 |
| ⑤USP | 「メンターが1名担当でつく。週の型（動画4日＋課題2日＋ワークショップ1日）が決まってるから迷わない」 |
| ⑥シーン | 「子どもが熱を出した週は、メンターに言えばペースを組み直せた」（実例として） |
| ⑦ビフォーアフター | 「2年動けなかった私が、2ヶ月後に最初の案件に応募してた。金額より、迷子が終わったことが大きい」 |
| ⑧クロージング | 「独学を続けるか迷ってる人は、無料勉強会で学習計画だけでも聞いてみてください」 |

### 台本例3：A-3「ママは毎日何してるの?」（認知・共感軸／Tier2潜在層・売り込みなし）

| ブロック | 内容 |
|---|---|
| ①フック | 「『ママは毎日何してるの?』子どもに聞かれて、固まった」 |
| ②現実の言語化 | 「家事して、送迎して、名もない仕事を100個やってる。のに、"何もしてない"気がする」 |
| ③教育 | 「実はこれ、暇なんじゃなくて**"自分の名前でやってることがない"**だけ」 |
| ④〜⑤ | この企画では商品ブロックを**意図的にスキップ**（潜在層向けは売らない。④⑤を飛ばし②③を厚くするのも標準運用として許容する） |
| ⑥〜⑦ | 「同じ気持ちだったママが、1年後『ママはお仕事してるんだよ』って言えるようになった話、またします」（次回予告型） |
| ⑧クロージング | CTAはフォローのみ：「同じ気持ちの人は、また見にきてください」。コメント誘発：「名もない仕事、あなたは1日いくつやってますか」 |

**共通運用ルール**：
- 必須要素8点（タイトル・秒数構成・セリフ・テロップ・撮影指示・表情・BGM・CV導線）を台本フォーマットに常設
- 1〜3秒フック最優先／1カット1〜2秒／不要な間は0.1秒単位でカット／三脚・ライト必須
- 投稿直後に自分で1番目のコメントを書き、コメント誘発文（「他にもあれば教えてください」型）を入れる
- 収入実例を出すときは必ず「個人の実例」「時期・職種・稼働時間」を添える（→§10）

## 8. 卒業生ストーリー（UGC）の型化と収集・活用

skimama最大のコンテンツ資産は卒業生の実例。**「良い声が来るのを待つ」のではなく、型を渡して質を揃えて収集する**。

### 8-1 ストーリーの標準型（バイアス→理想＋4点セット）

```
① ビフォー（バイアス状態）：入会前の状況・自己認識（「PCが苦手」「私に何もない」等、本人の言葉で）
② きっかけ：何を見て・何が最後の一押しだったか（→マーケの学習データにもなる）
③ 過程のリアル：挫折しかけた瞬間とメンター・仲間の関わり（きれいごとにしない）
④ アフター：職種・働き方の変化＋【最重要】情緒変化（「私にもできた」「子どもに言えるようになった」）
```

### 8-2 収集の仕組み（運用フロー案）

1. **収集タイミングの定点化**：初案件獲得時・卒業時・卒業6ヶ月後の3点でCS/メンター経由のアンケート＋インタビュー打診（要：社内オペレーション設計）
2. **インセンティブ**：インタビュー協力者への特典（コミュニティ内での紹介・発信サポート等。金銭対価にする場合はステマ規制・PR表記の整理が必要 →§10）
3. **同意と表記**：顔出し可否・実名/仮名・収入数値の公開範囲を書面で確認。本人の投稿を引用する場合も都度許諾
4. **シグナリング装置**：卒業生が「skimama卒」を名乗りたくなる仕掛け（修了証・プロフィール用テンプレート・卒業生タグ）を用意し、自発的UGCを構造化する

### 8-3 二次活用マップ（1素材→多面展開）

| 素材 | 展開先 |
|---|---|
| ロングインタビュー（60分収録） | YouTube長尺1本／リール・TikTok切り抜き3〜5本／カルーセル1本／Threads引用ポスト／LP・LINE配信の証拠枠／広告クリエイティブ素材 |
| 卒業生本人のSNS投稿 | 公式でのリポスト（許諾つき）／「卒業生の今」シリーズ化 |

## 9. 測定・改善のPDCA設計

**P（仮説）に最も時間を使う**（PDCAで最重要なのはP）。企画時点で「どのセル・どのJTBD・どの先行KPIを狙うか」を書いてから作る。

| サイクル | 見る指標 | 判断・アクション |
|---|---|---|
| 投稿ごと | 維持率カーブ（どの秒で離脱したか）・保存率・コメント数・プロフィールアクセス率 | 離脱点＝フック/テンポ問題。保存低＝情報価値不足。コメント低＝誘発設計不備 |
| 週次 | 軸別・PF別の先行KPI平均、ベスト/ワースト投稿の要素分解 | なぜ伸びたかの**要素**分析で勝ちパターンを抽出し翌週に反映 |
| 月次 | プロフィール→LINE遷移数、LINE登録数（Organic経由）、9セルの偏り | 遷移率が低ければ受け皿（プロフィール・ハイライト・特典訴求）を先に直す |
| 四半期 | 最終KPI・配分ルールの見直し・フェーズ移行判定 | ゲート条件方式：「維持率◯%以上の投稿が月◯本」「LINE登録月◯件」等の定量基準を満たしたら次Phase（広告連携拡大等）へ |

**「伸びない」ときの診断順序**：
1. 初動10〜20本以内か？ → ジャンル認識前。正常。継続
2. ジャンルが混在していないか？ → 複数ジャンル投稿は評価分散で伸びない
3. フック・テンポ・画質は基本原則を満たすか？ → クオリティ問題として個別改善
4. それでも変わらなければ企画の要素分析からやり直し（表面的な企画模倣ではなく「なぜ伸びるか」の要素へ）

## 10. 表現コンプライアンス（景表法・特商法・ステマ規制）

無形商材は成果に個人差があり、断定・保証表現は法令リスクかつブランド毀損（「怪しい副業」側に見られる）。**全投稿・全台本の公開前チェックリスト**とする。

| 使わない表現 | 言い換えの型 |
|---|---|
| 「必ず稼げる」「誰でも月◯万円」 | 「卒業生の◯%が在宅ワークを獲得（※算出定義を確認の上、注記併載）」「◯さんは卒業◯ヶ月で△△の案件を獲得（個人の実例）」 |
| 「絶対」「確実に」「保証します」 | 「仕事獲得まで伴走します」「実例をもとにお話しします」 |
| ビフォーアフターの収入を一般化（「主婦でもこれだけ稼げる」） | 「◯◯さんの場合：職種・稼働時間・時期を明記した個別ストーリー」 |
| 他社の名指しネガティブ比較 | 「スクール選びで見るべき観点」として基準を提示し、判断は視聴者に委ねる |
| 焦らせ系の虚偽限定（「残り◯名」が事実でない場合） | 実際の定員・期限がある場合のみ、事実として記載 |

- 実績数値（96.0%／99.3%／93.0%）は**算出定義・母数・調査時期の社内確認が済むまで、注記なしの単独掲出をしない**
- 卒業生に対価（金銭・特典）を渡して発信してもらう場合は**ステマ規制対象**。「PR」等の関係性明示を徹底
- 社内の表現ガイドライン有無を確認し、存在すればそちらを優先
